# Синапс C#

Отдельная версия игры на **C# / Blazor WebAssembly / .NET 10**. Игровая логика, модели уровней, исполнитель блоков, сравнение результатов, сюжетный прогресс и редактор уровней написаны на C#. Пользовательский интерфейс описан Razor-компонентами и CSS.

## Как открыть проект

Откройте `Synapse.Blocks.slnx` в Visual Studio или всю папку `SynapseCSharp` в JetBrains Rider / VS Code.

Для запуска из терминала:

```powershell
dotnet run
```

Сайт открывается по HTTP: `http://localhost:5023`. HTTPS специально не используется, поэтому Firefox не должен показывать ошибку сертификата.

## Как устроена проверка

У уровня нет заранее зашитой «правильной цепочки». Он задаёт:

- описание задания;
- объяснение входных и выходных данных;
- доступные типы блоков;
- любое количество тестов;
- вход и ожидаемый выход каждого теста.

Собранная игроком программа запускается отдельно на всех тестах. Уровень пройден, когда фактический результат блока «Выход» совпал с ожидаемым результатом во всех открытых и скрытых проверках.

## Где менять код

- `Models/GameModels.cs` — модели уровней, тестов, блоков и связей.
- `Services/BlockProgramRunner.cs` — исполнитель графа и команды блоков.
- `Services/LevelStore.cs` — сохранение, импорт и экспорт уровней.
- `Data/SeedLevels.cs` — 18 сюжетных операций кампании.
- `Pages/Home.razor` — игровое поле.
- `Pages/Landing.razor` — главная страница и изометрический Город Ноль.
- `Pages/LevelEditor.razor` — редактор уровней.
- `wwwroot/css/app.css` — весь дизайн.

## Команды блоков

В блоке «Операция» действие выбирается из выпадающего меню. Для арифметики, установки значения и добавления текста рядом появляется отдельное поле значения. Внутри программы выбранное действие хранится в одном из следующих форматов:

- `+ 2`, `- 3`, `* 4`, `/ 2` — арифметика;
- `set готово` — заменить текущее значение;
- `set` с пустым полем — намеренно записать пустую строку, выбранное действие при этом не сбрасывается;
- `append -OK` — добавить текст;
- `count` — посчитать символы или элементы JSON.

Условие:

- `> 20`, `< 0`, `>= 10`, `== ключ`, `!= ошибка`;
- `contains ошибка`, `starts REQ`, `ends -OK`.

Цикл:

- `input` или `вход` — число повторов берётся из входного значения;
- `20` — собственное количество повторов.

Синяя точка «повтор» ведёт к первой команде тела цикла, а последняя команда возвращается в нижний порт «возврат». Верхний порт «выход» продолжает программу после завершения всех повторов.

Рабочую область можно перемещать, зажав левую кнопку мыши на пустой сетке и потянув поле в нужную сторону. Полосы прокрутки скрыты, кнопки `−` и `+` меняют масштаб схемы.

Кнопка «Отладка» готовит первый тест уровня, а «Следующий блок» выполняет программу по одному узлу. Панель показывает вход, текущее значение и описание шага; обработанные блоки и активная связь подсвечиваются прямо на схеме.

## Уровни

Встроенная кампания содержит 18 связанных сюжетных уровней по программированию, алгоритмам, кибербезопасности и заказам жителей. В редакторе уровней можно создавать, копировать, удалять и переставлять уровни, добавлять любое количество тестов и отмечать проверки скрытыми. Кнопка «Экспорт JSON» сохраняет всю кампанию в файл, а «Импорт JSON» загружает её обратно.

До экспорта данные хранятся только в локальном хранилище браузера.
